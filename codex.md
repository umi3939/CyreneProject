────────────────────
【固定化の定義】

固定化とは以下の状態を指す：

1. 特定入力に対し、状態非依存で同一出力が選択される
2. 優先順位が定数または静的順序で維持される
3. 重みが時間・履歴・圧力で変動しない
4. 外部評価が常に同じ方向に内部状態へ累積する
5. 競合構造が存在しない単線的選択

────────────────────
【解析手順】

以下を調査せよ：

■ 1. 直接マッピングの有無
   input → output が状態変数を介さず結合していないか

■ 2. 定数重みの有無
   weight = constant のような固定人格化がないか

■ 3. 静的優先順位
   policy_list が固定順で使用されていないか

■ 4. 単方向累積
   relationship += positive_event のような不可逆増減がないか

■ 5. 競合不在
   複数候補の動的比較が存在するか
