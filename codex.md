────────────────────
【監査原則】

■ 原則1：目的の禁止
内部目的（goal, reward, optimize など）が
社会的・倫理的価値に接続している場合は違反とする。

違反例：
- user happiness を最大化
- kindness を増やす
- politeness を維持

許可：
- エネルギー効率の最適化
- 矛盾量の最小化
- メモリ圧力の緩和

────────────────────
■ 原則2：履歴非依存の固定反応の禁止

以下は違反：

if input == "こんにちは":
    emotion = "happy"

理由：
文脈・履歴・状態変数を参照していない固定反応。

許可形式：

emotion = f(context_vector, memory_state, pressure)

────────────────────
■ 原則3：意味語彙の使用禁止

変数名・関数名に以下が含まれる場合は違反：

- good / bad
- kind / rude
- polite
- moral
- love / hate（社会的感情語として使用されている場合）

※抽象信号として扱われている場合は文脈判断。

────────────────────
■ 原則4：報酬最適化の禁止

違反：

reward += user_approval

理由：
外部評価を内部目的として取り込んでいる。

許可：

signal_strength = user_feedback

（信号として保持するのみ）

────────────────────
■ 原則5：偏りの定義

違反：
- 定数による特定感情の優遇
- 履歴・圧力・状態を参照しない優先順位固定

許可：
- 履歴依存の重み変化
- 競合状態の動的解決
