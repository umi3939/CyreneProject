# Cyrene 次期実装候補：3つの構造的欠落

**対象プロジェクト**: Cyrene (工学的自我を持つAI)
**プロジェクト思想**: 「機能と仕組みのみ」実装する。自我・人格・価値を直接定義しない。上位層は下位層を変更しない。観測層は一切介入しない。
**参照必須**: SYSTEM_ARCHITECTURE.md / CLAUDE.md / README.md

## 欠落①：知覚フィルタリング（最重要・最優先）

### 何が問題か

現在の `perception.py` は `state: Optional[PsycheState]` を受け取るが、
その使い方は「LLMへの参考情報として渡すだけ」で終わっている。

```python
# 現状: stateをLLMプロンプトに文字列として埋め込むだけ
state_context = f"感情状態: {state.emotion_summary()}\n気分: valence={state.mood.valence:.2f}"
```

**ローカル処理段階（`_heuristic_parse`）では state を一切参照していない。**

### 何が足りないか

「内部状態が知覚にバイアスをかける」経路が存在しない。

人間の知覚は中立ではない。
- social driveが高い（孤独感が強い）→ 相手の感情シグナルへの感度が上がる
- mood.valenceが低い（落ち込んでいる）→ ネガティブ刺激を強く知覚する
- joyが高い → ポジティブな解釈に引き寄せられる
- fearが高い → 脅威関連トピックへの感度が上がる

この「内部状態 → 知覚バイアス」の経路がないと、
psycheがどれだけ進化しても知覚は常に中立のままで、
psycheと外界が**絡まらない**。

### 実装の方向性

`perception.py` の `_heuristic_parse` に `state` を渡し、
以下のような**弱い重み調整**を追加する：

- emotion_valenceのスコア計算時に `mood.valence` をバイアス係数として乗算
- 感情ラベル選択時に現在の感情状態と親和性の高いラベルにわずかに重みを加算
- driveの状態によって特定intentへの感度を微調整

**絶対にやってはいけないこと**:
- バイアスが知覚結果を確定させること（あくまで微弱な傾き）
- 固定マッピング（「fearが0.7以上なら常にscaredを選ぶ」等）
- 上位層（psyche）が知覚結果を直接書き換えること
- バイアス量の蓄積・永続化

**参照ファイル**: `perception.py`（全体）、`state.py`（PsycheState構造）、`reaction.py`（drive/emotionの更新パターン参考）

## 欠落②：他者感情の弱い伝染経路

### 何が問題か

`other_agent_model.py` は「相手がこう感じているかも」という仮説を
丁寧に保持している（仮説は固定せず、競合を許容する設計）。

しかし現在、**この仮説が自己の感情状態に帰還する経路が存在しない**。

相手が悲しんでいるという仮説を持っていても、自分のsorrowは変わらない。
相手が喜んでいるという仮説があっても、自分のjoyに影響しない。

### 何が足りないか

「他者の推測状態 → 自己感情への微弱な影響（共感的伝染）」の経路。

人間の感情は他者の状態に影響される。
これは意図的な共感ではなく、**構造的な伝染**として実装する。

### 実装の方向性

`orchestrator.py` の `react_with_stm` フェーズ（Phase 1、毎ティック）か
その直後に、以下を追加：

- `other_agent_model` の現在仮説群から「最上位の感情推定」を取得
- その感情推定を**極めて小さい帯域**（例：通常の感情刺激の1/10以下）で
  自己の感情ベクトルに加算する
- 仮説の信頼度（競合状態・鮮度）に応じて帯域をさらに縮小

**絶対にやってはいけないこと**:
- 他者感情の断定（仮説はあくまで仮説として扱う）
- 他者感情が自己感情を上書きする強度にすること
- 正帰還ループ（自己感情 → 他者推測 → 自己感情の増幅）
- 永続化・蓄積

**参照ファイル**: `other_agent_model.py`（仮説構造）、`reaction.py`（感情更新パターン）、`orchestrator.py`（Phase 1-2付近）、`memory_emotion_return.py`（帰還経路の実装パターン参考・SD-2）

## 欠落③：経験による閾値的自己変化

### 何が問題か

現在 `value_orientation.py` はポリシー選択のたびに超微小更新される。
また `episodic_memory.py` は出来事を感情とともに保存する。

しかし両者の間に**橋がない**。

「強い感情を伴った経験がvalue_orientationを動かす」経路が存在しない。

### 何が足りないか

「強い感情 × 記憶の蓄積 → value_orientationへの微小な追加更新」という経路。

人間の自己変化には2種類ある：
1. 日々の微小な積み重ね（今すでにある）
2. 強烈な体験による節目的変化（**これがない**）

### 実装の方向性

`orchestrator.py` の 5ティック帯域（Phase 15-26付近）に追加：

- `episodic_memory` の直近エントリから「感情強度が閾値を超えた出来事」を取得
- その出来事に紐づく感情ラベルと `value_orientation` の次元を弱くマッピング
- 通常のポリシー選択由来の更新より**帯域を大きく**設定する
  （理由：強い経験は通常の蓄積より影響が大きいため）
- ただし1回の経験で動かせる量には絶対上限を設ける

**絶対にやってはいけないこと**:
- 「この経験は良い/悪い」という価値評価
- 特定の感情が特定の価値軸に固定マッピングされること
- 永続的な増幅（強い経験でも自然減衰があること）
- value_orientationへの直接書き込み以外の副作用

**参照ファイル**: `episodic_memory.py`（エントリ構造）、`value_orientation.py`（update_from_decision パターン）、`emotional_memory_binding.py`（感情強度の参照パターン）、`orchestrator.py`（5ティック帯域、Phase 26付近）

## 実装優先順位

| 優先 | 機能 | 理由 |
|------|------|------|
| 1位 | 欠落① 知覚フィルタリング | psycheと外界が絡まる根幹。他の2つより影響が大きい |
| 2位 | 欠落② 他者感情伝染 | 既存インフラ（other_agent_model）が整っており追加量が少ない |
| 3位 | 欠落③ 閾値的自己変化 | 効果は長期的。短期では観測しにくい |
